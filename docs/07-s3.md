### 1. Configuraci√≥n de S3 en Spring Boot

Primero, configuramos la conexi√≥n a S3 en nuestra aplicaci√≥n utilizando las credenciales de AWS y la regi√≥n correspondiente. Esto se hace en la clase `StorageConfig`:

```java
    private String accessKey;

    @Value("${cloud.aws.credentials.secretKey}")

    @Value("${cloud.aws.credentials.sessionToken}")

                .standard()
                .withRegion(String.valueOf(RegionUtils.getRegion(region)))
                .withCredentials(new AWSStaticCredentialsProvider(basicSessionCredentials))
                .build();
    }
}
```

### 3. Servicio de Almacenamiento: Subida de Archivos a S3

```java
public String uploadFile(MultipartFile file, String objectKey) throws Exception {
    if (file.isEmpty() || file.getSize() > 5242880)
        throw new IllegalArgumentException("Invalid file size");

    if (objectKey.startsWith("/") || !objectKey.contains("."))
        throw new IllegalArgumentException("Invalid object key");

    ObjectMetadata metadata = new ObjectMetadata();
    metadata.setContentType(file.getContentType());
    metadata.setContentLength(file.getSize());

    if (s3Client.doesObjectExist(bucketName, objectKey))
        deleteFile(objectKey);

    try (InputStream inputStream = file.getInputStream()) {
        s3Client.putObject(new PutObjectRequest(bucketName, objectKey, inputStream, metadata));
        return objectKey;
    } catch (IOException e) {
        throw new Exception("Failed to upload file to S3", e);
    }
}
```

**Puntos Clave:**
- **Validaci√≥n de Archivos**: Se valida el tama√±o del archivo y el formato del `objectKey` antes de proceder, evitando errores y asegurando que los archivos subidos cumplan con los requisitos.
- **Metadata del Archivo**: Se a√±aden metadatos como el tipo de contenido y la longitud del archivo, lo que es importante para un manejo correcto en S3.
- **Eliminaci√≥n de Archivos Duplicados**: Si ya existe un archivo con el mismo `objectKey`, se elimina antes de subir el nuevo archivo, asegurando que no haya conflictos.

### 4. Generaci√≥n de una URL Pre-firmada

#### üåê ¬øQu√© es una URL Pre-firmada (Presigned URL)?

Una **URL pre-firmada** (Presigned URL) es una URL que proporciona acceso temporal y seguro a un objeto almacenado en Amazon S3, sin necesidad de que el usuario tenga credenciales directas de AWS. La URL pre-firmada est√° "firmada" con credenciales de AWS y tiene una duraci√≥n limitada, despu√©s de la cual expira y deja de ser v√°lida.

##### ¬øC√≥mo funciona una Presigned URL?

Cuando generas una URL pre-firmada:

1. **Se firma la URL** con tus credenciales de AWS, lo que asegura que solo quienes tengan la URL puedan acceder al archivo.
2. **Se especifica un tiempo de expiraci√≥n**: Definir el tiempo de validez garantiza que el acceso sea temporal, aumentando la seguridad.
3. **Se establece el permiso**: Puedes definir si la URL permite operaciones de lectura, escritura o ambas.

Por ejemplo, en el contexto de una aplicaci√≥n web donde los usuarios pueden subir y ver fotos de perfil, una URL pre-firmada te permite:

- **Subir una imagen** de forma directa y segura al bucket S3.
- **Obtener la URL temporal** para que el usuario pueda visualizar su imagen de perfil sin exponer el archivo p√∫blicamente.

```java
public String generatePresignedUrl(String objectKey) {
    if (!s3Client.doesObjectExist(bucketName, objectKey))
        throw new RuntimeException("File not found in S3");

    Date expiration = new Date(System.currentTimeMillis() + 1000 * 60 * 60);
    GeneratePresignedUrlRequest generatePresignedUrlRequest =
            new GeneratePresignedUrlRequest(bucketName, objectKey)
                    .withExpiration(expiration);

    URL url = s3Client.generatePresignedUrl(generatePresignedUrlRequest);
    return url.toString();
}
```

### 1. Creaci√≥n de un Formulario de Carga de Archivos

## üìù ¬øQu√© es FormData y para qu√© se utiliza en el Frontend?

**FormData** es una interfaz proporcionada por JavaScript que te permite construir un conjunto de pares clave/valor, representando campos de un formulario, que pueden ser enviados f√°cilmente utilizando m√©todos como `fetch` o `XMLHttpRequest`. ### ¬øPara qu√© sirve FormData?

FormData es extremadamente √∫til en aplicaciones web donde los usuarios interact√∫an con formularios que contienen campos de texto y archivos. Algunas situaciones comunes donde se utiliza FormData son:

1. **Subida de archivos**: Facilita la transmisi√≥n de archivos (como im√°genes, documentos, etc.) desde el navegador al servidor, permitiendo al usuario adjuntar archivos a un formulario y enviarlos con otros datos.
  
2. **Env√≠o de datos complejos**: Cuando un formulario contiene m√∫ltiples campos de diferentes tipos (texto, selectores, casillas de verificaci√≥n, etc.), FormData puede manejar todo ese contenido y enviarlo de una sola vez al servidor.

3. **Interacciones asincr√≥nicas**: FormData es ideal para aplicaciones modernas que requieren enviar datos al servidor sin recargar la p√°gina, mejorando la experiencia del usuario.

### ¬øC√≥mo funciona FormData?

El uso de FormData en el frontend es sencillo y se hace en tres pasos principales:

1. **Creaci√≥n del objeto FormData**: Puedes crear un objeto FormData vac√≠o o inicializarlo con un formulario HTML existente.
   

2. **A√±adir campos y archivos**: Puedes agregar datos al objeto FormData usando el m√©todo `.append()`. Este m√©todo permite agregar tanto campos de texto como archivos.


3. **Enviar el FormData al servidor**: Finalmente, el objeto FormData se env√≠a al servidor utilizando `fetch` o `XMLHttpRequest`.

   ```javascript

       method: 'POST',
   ```

Ahora s√≠, con esta informaci√≥n en mente, podemos proceder a crear el formulario de carga de archivos en React:

```tsx
import React, { useState } from 'react';
import axios from 'axios';

const FileUpload = () => {
    const [file, setFile] = useState<File | null>(null);

    const handleFileChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        const selectedFile = e.target.files?.[0];
        if (selectedFile) setFile(selectedFile);
    };

    const handleUpload = async () => {
        if (!file) return;

        const formData = new FormData();
        formData.append('file', file);

        try {
            await axios.post('http://localhost:8080/api/profile-pic', formData, {
                headers: {
                    'Content-Type': 'multipart/form-data'
                }
            });
            alert('File uploaded successfully');
        } catch (error) {
            alert('Failed to upload file');
        }
    };

    return (
        <div>
            <input type="file" onChange={handleFileChange} />
            <button onClick={handleUpload}>Upload</button>
        </div>
    );
};

export default FileUpload;
```

**Explicaci√≥n:**
- **Estado Local**: Utilizamos el estado local de React para almacenar el archivo seleccionado por el usuario.
- **Manejo de Eventos**: Los eventos `onChange` y `onClick` se utilizan para capturar la selecci√≥n del archivo y la carga del archivo, respectivamente.
- **Env√≠o de Archivos**: Al hacer clic en el bot√≥n "Upload", el archivo se env√≠a al backend a trav√©s de una solicitud POST.
- **Manejo de Errores**: Se muestra una alerta en caso de que la carga del archivo falle.
- **Nota**: Aseg√∫rate de que la URL de la API en `axios.post` coincida con la URL de tu backend.
